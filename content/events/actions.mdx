# Actions

Actions perform tasks within workflows and can be triggered by events or executed by agents.

## Overview

The Actions collection provides a way to define, manage, and execute operations in your AI applications. Actions can:

- Perform specific tasks within workflows
- Be triggered by events
- Be executed by agents
- Integrate with external systems and APIs

## Key Features

- **Task-Oriented**: Actions represent specific tasks that can be performed
- **Reusable**: Actions can be used across multiple workflows and agents
- **Composable**: Actions can be combined to create complex operations
- **Monitored**: Track action execution and performance

## Defining Actions

Actions can be defined using the Events.do API or through the dashboard interface.

```typescript
// Example action definition
const SendEmailAction = {
  name: 'SendEmailAction',
  description: 'Sends an email to a specified recipient',
  input: z.object({
    to: z.string().email(),
    subject: z.string(),
    body: z.string(),
    attachments: z.array(z.object({
      name: z.string(),
      content: z.string(),
      contentType: z.string()
    })).optional(),
    cc: z.array(z.string().email()).optional(),
    bcc: z.array(z.string().email()).optional(),
    template: z.string().optional(),
    templateData: z.record(z.any()).optional()
  }),
  output: z.object({
    success: z.boolean(),
    messageId: z.string().optional(),
    error: z.string().optional()
  }),
  execute: async (input) => {
    // Implementation logic
    return { success: true, messageId: 'msg-123' }
  },
  rateLimit: {
    limit: 100,
    period: '1m'
  },
  retry: {
    maxAttempts: 3,
    backoff: {
      type: 'exponential',
      initialDelay: 1000
    }
  }
}
```

## Action Categories

Actions.do supports various action categories:

### Communication Actions

Send messages through different channels:

```typescript
const SendSlackMessageAction = {
  name: 'SendSlackMessageAction',
  input: z.object({
    channel: z.string(),
    message: z.string(),
    blocks: z.array(z.any()).optional()
  }),
  // Implementation...
}
```

### Data Actions

Manipulate data in your systems:

```typescript
const UpdateDatabaseAction = {
  name: 'UpdateDatabaseAction',
  input: z.object({
    collection: z.string(),
    id: z.string(),
    data: z.record(z.any()),
    options: z.object({
      upsert: z.boolean().optional()
    }).optional()
  }),
  // Implementation...
}
```

### Integration Actions

Connect with external systems:

```typescript
const CallExternalAPIAction = {
  name: 'CallExternalAPIAction',
  input: z.object({
    url: z.string().url(),
    method: z.enum(['GET', 'POST', 'PUT', 'DELETE']),
    headers: z.record(z.string()).optional(),
    body: z.any().optional(),
    timeout: z.number().optional()
  }),
  // Implementation...
}
```

## Using Actions

Actions can be executed throughout your AI applications:

```typescript
// Execute an action directly
const result = await actions.execute('SendEmailAction', {
  to: 'user@example.com',
  subject: 'Welcome to our platform',
  body: 'Thank you for signing up!',
  template: 'welcome-email',
  templateData: {
    userName: 'John Doe',
    activationLink: 'https://example.com/activate/abc123'
  }
})

// Use in workflows
const userOnboardingWorkflow = {
  steps: [
    {
      id: 'createAccount',
      function: 'createUserAccount',
      input: { userData: '{{input.userData}}' }
    },
    {
      id: 'sendWelcome',
      action: 'SendEmailAction',
      input: { 
        to: '{{input.userData.email}}',
        subject: 'Welcome to our platform',
        template: 'welcome-email',
        templateData: {
          userName: '{{input.userData.name}}',
          activationLink: '{{createAccount.activationLink}}'
        }
      }
    }
  ]
}
```
